# BDD Examples Guide

## Where to demo
- Scenario quality comparison:
  - `src/test/resources/features/training/bdd/scenario-quality-comparison.feature`
- Background + Rule example:
  - `src/test/resources/features/training/bdd/background-and-rule-example.feature`

## What to teach from these files
- Same intent, three writing styles:
  - poorly written scenario (anti-example)
  - improved explicit scenario
  - non-outline duplicate scenario
  - Scenario Outline improved version
- Reusable setup via `Background`
- Business rules via `Rule`

## Trainer talking points
- Good BDD keeps intent clear, not technical.
- Steps should read like business behavior.
- Scenario Outline is helpful only when repetition is meaningful.
- Anti-examples are for refactoring exercises, not implementation templates.
